# Qwen3.5 GGUF Head-to-Head Benchmark

Compares **Qwen3.5-4B vs Qwen3.5-9B** across GGUF quantization levels on a single
NVIDIA RTX A5000 (24 GB), using `llama.cpp`. Measures three axes:

- **Speed** — prompt-processing (pp) and token-generation (tg) throughput via `llama-bench`.
- **VRAM** — loaded GPU memory footprint.
- **Quality** — a reasoning probe (GSM8K/AIME-style items with known answers) and an
  **agentic tool-call validity** probe (does the model emit a valid, correct tool call?).

## Quant ladder

| Quant | Purpose |
|-------|---------|
| `Q8_0` | Near-lossless baseline (quality ceiling) |
| `Q4_K_M` | Standard on-device 4-bit sweet spot |
| `UD-Q4_K_XL` | Unsloth Dynamic (imatrix) 4-bit — tests whether imatrix recovers the Q8→Q4 drop |

## Layout

- `probes.py` — reasoning + tool-call probe datasets.
- `run_bench.py` — orchestrator: runs speed, VRAM, reasoning, and tool-call evals per model.
- `results.json` — output metrics (generated by a run).

## Setup

```bash
python3 -m venv .venv && . .venv/bin/activate
pip install huggingface_hub hf_transfer

# Build llama.cpp with CUDA (Ampere = arch 86)
git clone --depth 1 https://github.com/ggml-org/llama.cpp.git
cmake -S llama.cpp -B llama.cpp/build -DGGML_CUDA=ON -DCMAKE_CUDA_ARCHITECTURES=86
cmake --build llama.cpp/build --config Release -j

# Download GGUF quants
hf download unsloth/Qwen3.5-4B-GGUF Qwen3.5-4B-Q4_K_M.gguf --local-dir models/q4b
# ... (see run_bench.py MODELS list for the full set)

python run_bench.py
```
